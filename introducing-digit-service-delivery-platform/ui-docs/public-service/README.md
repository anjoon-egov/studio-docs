# Public Service

**Objective:** <mark style="color:red;">This</mark> <mark style="color:red;">module consumes configurations generated from the</mark> <mark style="color:red;"></mark><mark style="color:red;">**Process Flow Designer**</mark> <mark style="color:red;"></mark><mark style="color:red;">in the Service Designer. It dynamically generates the</mark> <mark style="color:red;"></mark><mark style="color:red;">**Apply**</mark><mark style="color:red;">,</mark> <mark style="color:red;"></mark><mark style="color:red;">**Inbox**</mark><mark style="color:red;">,</mark> <mark style="color:red;"></mark><mark style="color:red;">**Search**</mark><mark style="color:red;">, and</mark> <mark style="color:red;"></mark><mark style="color:red;">**View**</mark> <mark style="color:red;"></mark><mark style="color:red;">screens that together form the complete user-facing workflow of a service. This ensures that any service created and published through the Service Designer becomes instantly usable for end users and employees without additional coding.</mark>

## About Module

The **Public Service** module interprets the service configuration JSON (containing form structure, workflow, and actions) to automatically render the application flow:

1. **Apply Screen:** Allows citizens or employees to submit new service applications.
2. **Inbox Screen:** Lists all existing applications assigned to a user or department, along with their workflow status.
3. **Search Screen:** Enables users to search for applications based on configured parameters (like Application Number, Mobile Number, or Applicant Name).
4. **View Screen:** Displays complete details of an application, including workflow status, documents, and permitted actions.

This metadata-driven approach ensures:

* Consistency between design and runtime experience.
* Faster rollout of new services.
* Role-based screen rendering for secure access control.
